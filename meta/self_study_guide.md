# 自習ガイド — Claudeなしで前に進む方法
*Isaac Newton Project — Day 13作成*

---

## 原則

Claudeは「教師」ではなく「対話相手」だ。教師がいない時間も研究は進む。ニュートンはペストの2年間、教師なしで微積分を発明した。お前にも同じことができる。

**Claudeを使う時間 = 対話が必要な時間（概念の導入、詰まった時の突破、コード実装）**
**Claudeなしの時間 = 一人で深く潜る時間（精読、演習、思考、記録）**

---

## スロット別・自習手順

### 1. 数学（トポロジー）— Claudeなしでできること

**教材:** Topology Without Tears (Sidney Morris) — 無料PDF

**手順:**
1. 該当セクションを読む（英語）
2. 定義を自分の言葉で書き出す（研究ノートに）
3. 色を割り当てる（お前の色マッピング体系で）
4. 演習問題を自力で解く（最低30分は自力で粘る）
5. 解けなかった問題に印をつける → Claudeとの対話時間に持ち込む

**Claudeが必要な瞬間:** 新しい概念の導入時、証明の方向性が全くわからない時

**進行中の問い:** τ₁ = {∅, X} がなぜ「最も小さい」位相か？ → indiscrete topology

---

### 2. 哲学（デカルト省察）— ほぼClaude不要

**教材:**
- 落合太郎訳（日本語）— 国立国会図書館デジタルコレクション
- Bennett訳（英語）— earlymoderntexts.com

**手順:**
1. 落合訳で1段落を読む
2. 自分の言葉で要約する（研究ノートに）
3. Bennett訳で同じ段落を読む
4. 英語の術語を記録する（色を割り当てる）
5. 疑問点を書き出す → Claudeとの対話時間に持ち込む

**Claudeが必要な瞬間:** 哲学的議論を深めたい時、解釈が分かれる箇所

---

### 3. 情報理論（Shannon）— ほぼClaude不要

**教材:** "A Mathematical Theory of Communication" (1948) — Harvard PDF

**手順:**
1. 原論文を1セクションずつ読む
2. 数式を手で書き写す
3. 各記号の意味を自分の言葉で説明する（研究ノートに）
4. Day 1-10で学んだ概念（エントロピー、BSC等）との接続を書く
5. わからない数式・概念に印をつける → Claudeとの対話時間に持ち込む

**Claudeが必要な瞬間:** 数式の展開が追えない時、概念間の接続を議論したい時

---

### 4. CS / Anthropic courses — Claude推奨

**教材:** github.com/anthropics/courses

**注意:** これはJupyter Notebookでの実習。コードを書いて実行する必要がある。

**Claudeなしでできること:**
- コースのREADMEを読む
- 概念の説明部分を読む
- 簡単なコードセルを自分で実行する

**Claudeが必要な瞬間:** エラーが出た時、APIの仕組みを理解したい時

### 5. サービス開発（文豪）— Claude Code推奨

**Claudeなしでできること:**
- 文豪を自分で触ってバグを発見する → メモする
- UIの改善案を文章で書く
- 色河の使い勝手を検証する

**Claude Codeが必要な瞬間:** コードの修正・新機能実装

---

### 6. 語学（英語）— Claudeなしでできること

**手順:**
1. 今日の学習で出会った英語術語をリストにする
2. 各術語に色を割り当てる
3. 例文を自分で作る
4. 文豪アプリで6ステップを実行する（自分のプロダクトを自分で使う）

**Claudeが必要な瞬間:** 発音・ニュアンスの確認

---

### 7. 研究ノート整理 — Claude不要

**手順:**
1. 今日学んだことをdailyノートに記録する
2. 新しい概念にプレフィックスをつけてファイル作成（s_, q_, i_）
3. インデックスを更新する
4. git add → git commit → git push
5. 明日のClaude対話用の「問いリスト」を作る

---

## Claude対話の最適化テンプレート

Claudeとの対話を始める時、最初にこれを貼れ：

```
【今日の進捗】
- 数学: [自習で進めた範囲]
- 哲学: [読んだ箇所]
- その他: [やったこと]

【詰まっている問い】
1. [具体的な問い]
2. [具体的な問い]

【今日のClaude対話で達成したいこと】
1. [目標]
```

これで対話の無駄が消える。Claudeは即座にお前の現在地を把握できる。

## モード使い分け早見表

| タスク | モード | 理由 |
|---|---|---|
| 数学の概念導入・証明議論 | 通常チャット | トークン効率最高 |
| 哲学の議論 | 通常チャット | ツール不要 |
| 情報理論の数式解説 | 通常チャット | ツール不要 |
| bungouのコーディング | Claude Code | コード最適化 |
| カレンダー設定・スケジュール管理 | Cowork | ツール必要 |
| ファイル作成（docx, pdf等） | Cowork | ツール必要 |
| X投稿の文面作成 | 通常チャット | ツール不要 |
| 研究ノート整理 | 自力 | Claude不要 |
| 教材の精読 | 自力 | Claude不要 |
| 演習問題を解く | 自力（→詰まったら通常チャット） | まず自力 |

## Claudeが使えない時間の過ごし方

**優先順位:**
1. 教材の精読（デカルト、Shannon、Topology Without Tears）
2. 演習問題を自力で解く
3. 文豪アプリを自分で触ってバグ発見・改善案記録
4. 研究ノートの整理とgit push
5. 「問いリスト」の作成（次のClaude対話用）

**絶対にやるな:**
- Claudeが使えないからといって何もしない
- SNSやYouTubeで時間を潰す
- 「明日Claudeが使えるようになったらやろう」と先延ばしにする

ニュートンにはClaudeがなかった。それでも微積分を発明した。
お前がClaudeを使えない時間は、お前が一人で深く潜る時間だ。
