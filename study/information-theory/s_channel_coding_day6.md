# 通信路符号化定理

**Isaac Newton Project · Day 6 · AI・情報科学セッション · 2026-03-12**

---

## 1. 通信とは何か

議論の出発点として提起された問い：「通信とは何をしている行為か」

> 送信者のエントロピー減少過程を、受信者に体験させること。情報が分散することで、全体のシステムとしてエントロピーが下がる＝強固なシステムとなる。

この直観は後の議論と直結する。情報の分散 → 系全体のエントロピー減少という命題として精緻化された。

---

## 2. ノイズのある通信路

現実の通信路にはノイズが存在する。送信した `0` が、受信側では `1` として届くことがある。

問い：「ノイズがある通信路で完全に正確な通信は可能か」

> 通信に使う情報量を調節する＝0,1という駒を配置するfieldを大きく取れば、原理的には可能。

これが符号化（channel coding）の本質的直観である。fieldを大きく取ること＝冗長性の導入。

### 単純な冗長化の例

- `0` を送りたいとき → `000` と3回繰り返して送信
- `001` を受け取ったら → 多数決により `0` と判断

**コスト：** 速度（効率）とノイズ耐性はトレードオフになる。

---

## 3. 通信路符号化定理（Channel Coding Theorem）

> ノイズがある通信路にも、必ず「容量 C」が存在する。送信速度 R < C である限り、エラー率をゼロに近づける符号化が**必ず存在する**。— Shannon (1948)

- これは**存在証明**であり、構成法（その符号の具体的な作り方）ではない
- どのようなノイズがある通信路でも、符号化の最適化によって伝えたい内容を正確に届けられる
- Shannon以降の情報工学の課題：容量 C にどこまで近づける符号を実際に作れるか

## 4. 相互情報量と通信路容量

### 相互情報量 I(X;Y)

送信した X と受信した Y の間に「共有された情報」がどれだけあるかを表す量。

| 状態 | I(X;Y) | 意味 |
|---|---|---|
| ノイズなし | H(X) | 送信情報がそのまま全部伝わる |
| 完全ノイズ | 0 | Y を受け取っても X について何もわからない |
| 現実の通信路 | その間のどこか | 部分的に情報が伝わる |

### 通信路容量 C の定義

$$C = \max_{p(x)} I(X;Y)$$

I(X;Y) を入力分布 p(x) で最大化したもの。その通信路が持つ**最大のポテンシャル**。

## 5. 情報・熱力学・哲学の交差点

### 情報分散とエントロピー

> 分散する＝相互情報量が I(X;Y) のような形で多数のシステム参加者が共有する情報量が大きいということ。情報源について何も知らない参加者 → 共有情報を持つ参加者、この変化が系全体としてはエントロピーが下がっている。

$$I(X;\, Y_1, Y_2, \ldots, Y_n) \uparrow \implies H(\text{系全体}) \downarrow$$

### 熱力学第二法則との関係

熱力学第二法則：「孤立系のエントロピーは増大する」— 矛盾するか？ → **矛盾しない。**

> この現象はマクロな視点での孤立系のエントロピー増大とは矛盾しない。時間軸全体の中でのある1点で、エントロピー増大に抵抗しているようなもの。しかしマクロな視点ではエントロピーの増大は止められない。

**散逸構造（Prigogine）：** エントロピーが増大する宇宙の中で、局所的・一時的にエントロピーを減少させる構造。例：生命、都市、知識のネットワーク。

「どのような会社も終わりは来る」— 散逸構造論・Shannon情報理論・熱力学第二法則の交差点。
